    DFS vs BFS：算法世界的“深度”与“广度”，你该如何抉择？ #

    引言 #

    在算法的江湖里，DFS 和 BFS 如同两位性格迥异的绝世高手。他们师出同门——都属于图（或树）的遍历算法，但招式却大相径庭。DFS（深度优先搜索）是一位探索家，不撞南墙不回头；而 BFS（广度优先搜索）则像一位指挥官，稳扎稳打，层层推进。

    很多初学者（包括我当初）常常会疑惑：什么时候该请出“探索家”DFS，什么时候又该派出“指挥官”BFS 呢？

    别担心，这篇博客将通过生动的比喻、清晰的图解和实战代码，带你彻底搞懂它们，让你在未来的算法之路上，面对任何遍历问题都能胸有成竹。

    什么是深度优先搜索 (DFS)? - “一条路走到黑”的探索家 #

    DFS 的核心思想非常符合人类的直觉：一条路走到黑。想象一下你在走一个巨大的迷宫，为了找到出口，你决定采用以下策略：

    1. 随便选一条没走过的岔路。
    2. 沿着这条路一直走下去，直到遇到死胡同或者回到已经走过的路口。
    3. 这时，退回到上一个岔路口，再选一条你没试过的路继续探索。
    4. 重复以上步骤，直到找到出口或者所有路都试完。

    这就是 DFS！它会尽可能深地探索一个分支，然后再回溯。

    核心数据结构 #

    DFS 依赖于 栈 (Stack) 的思想。当我们用 递归 (Recursion) 来实现 DFS 时，其实就是巧妙地利用了程序的调用栈。

    Go 代码模板 (递归) # 

    // visited 用来记录访问过的节点，防止死循环
var visited = make(map[Node]bool)

func dfs(node Node) {
    // 终止条件 (根据题目定义)
    if node == nil {
        return
    }

    // 标记已访问
    visited[node] = true
    // 处理当前节点逻辑 (例如：打印)
    fmt.Println(node.Value)

    // 递归访问邻居
    for _, neighbor := range node.Neighbors {
        if !visited[neighbor] {
            dfs(neighbor)
        }
    }
}

    什么是广度优先搜索 (BFS)? - “稳扎稳打”的辐射源 #

    BFS 则完全不同，它的策略是 层层推进。想象一下在平静的湖面投下一颗石子，水波纹是如何扩散的？它会先触及最近的一圈，再扩散到稍远的一圈，一圈一圈，非常规律。

    BFS 就是这样，它从起点开始，先访问所有距离为 1 的邻居，然后访问所有距离为 2 的邻居，以此类推。

    同样是上面那张图，从节点 A 开始，BFS 的探索路径是这样的：

    核心数据结构 #

    BFS 的实现离不开 队列 (Queue)。队列 “先进先出” (FIFO) 的特性完美保证了算法的“分层”探索。

    Go 代码模板 # 

    func bfs(startNode Node) {
    if startNode == nil {
        return
    }

    queue := list.New() // 使用 Go 的 list 包模拟队列
    visited := make(map[Node]bool)

    // 起点入队并标记
    queue.PushBack(startNode)
    visited[startNode] = true

    for queue.Len() > 0 {
        // 出队
        element := queue.Front()
        node := element.Value.(Node)
        queue.Remove(element)

        // 处理当前节点逻辑
        fmt.Println(node.Value)

        // 将未访问的邻居入队
        for _, neighbor := range node.Neighbors {
            if !visited[neighbor] {
                queue.PushBack(neighbor)
                visited[neighbor] = true
            }
        }
    }
}

    核心对决：我该用哪个？ #

    理论讲完了，现在来看最关键的问题：什么情况下用谁？

    派出 BFS 的场景：寻找最短路径 #

    当你看到题目中出现 “最短”、“最少”、“最小步数” 等关键词，并且图中每条边的权重都为 1（或者说没有权重），那么请毫不犹豫地选择 BFS！

    • 原因：BFS 按层遍历的特性，保证了它第一次找到目标节点时，所经过的路径一定是层数最少的，也就是最短的。
    • 经典例题：
      1. LeetCode 994. 腐烂的橘子：求所有橘子都腐烂的最短时间。这就是一个典型的分层问题，每一分钟就是一层。
      2. LeetCode 111. 二叉树的最小深度：找到根节点到最近叶子节点的最短路径。
      3. 走迷宫求最少步数：从起点到终点，最少需要走几步。

    派出 DFS 的场景：探索所有可能性 #

    当题目不要求最优解，而是问你 “有没有”、“能不能”，或者需要你找出 “所有” 的解决方案时，DFS 就是你的不二之选。

    • 原因：DFS 的递归结构天然适合进行深度探索和回溯，非常适合解决需要“试探”所有可能性的问题。
    • 经典例题：
      1. LeetCode 200. 岛屿数量：判断图中有多少个连通块。只需从一个 ‘1’ 开始 DFS，就能把整个岛屿“沉没”（标记为已访问），然后计数即可。
      2. LeetCode 46. 全排列：找出数字的所有排列组合。这本质上就是通过 DFS 遍历一棵“决策树”。
      3. 组合总和、N皇后、解数独：这些都是经典的回溯问题，而回溯算法就是 DFS 的一种应用。

    总结与速查表 #

    如果还觉得有点模糊，没关系，收好下面这张速查表：

    特性 深度优先搜索 (DFS) 广度优先搜索 (BFS)
    核心思想 一条路走到黑，然后回溯 一层一层地向外辐射
    数据结构 栈 (Stack) 或 递归 (Recursion) 队列 (Queue)
    关键应用 连通性、路径搜索、回溯、找出所有解 无权图最短路径、层序遍历
    找到的解 不保证最优 保证最短/最优 (在无权图中)
    代码实现 递归写法简洁，非递归需要手动维护栈 需要手动维护队列
    一句话 不撞南墙不回头 稳扎稳打，地毯式搜索

    结语 #

    DFS 和 BFS 没有绝对的优劣之分，它们是解决不同问题的两种强大工具。理解了它们的本质思想和适用场景，就像掌握了两种上乘心法。

    希望这篇博客能帮助你彻底厘清它们的脉络。在未来的算法之路上，无论是选择深邃的 DFS，还是稳健的 BFS，你都能胸有成竹，游刃有余。
