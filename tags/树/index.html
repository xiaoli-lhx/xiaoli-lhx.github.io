    DFS vs BFS：算法世界的“深度”与“广度”，你该如何抉择？

    October 5, 2025
    DFS, BFS, 图论, 树, 队列, 栈, 算法思想

    DFS vs BFS：算法世界的“深度”与“广度”，你该如何抉择？ #

    引言 #

    在算法的江湖里，DFS 和 BFS 如同两位性格迥异的绝世高手。他们师出同门——都属于图（或树）的遍历算法，但招式却大相径庭。DFS（深度优先搜索）是一位探索家，不撞南墙不回头；而 BFS（广度优先搜索）则像一位指挥官，稳扎稳打，层层推进。

    很多初学者（包括我当初）常常会疑惑：什么时候该请出“探索家”DFS，什么时候又该派出“指挥官”BFS 呢？

    别担心，这篇博客将通过生动的比喻、清晰的图解和实战代码，带你彻底搞懂它们，让你在未来的算法之路上，面对任何遍历问题都能胸有成竹。

    什么是深度优先搜索 (DFS)? - “一条路走到黑”的探索家 #

    DFS 的核心思想非常符合人类的直觉：一条路走到黑。想象一下你在走一个巨大的迷宫，为了找到出口，你决定采用以下策略：

    1. 随便选一条没走过的岔路。
    2. 沿着这条路一直走下去，直到遇到死胡同或者回到已经走过的路口。
    3. 这时，退回到上一个岔路口，再选一条你没试过的路继续探索。
    4. 重复以上步骤，直到找到出口或者所有路都试完。

    这就是 DFS！它会尽可能深地探索一个分支，然后再回溯。

    核心数据结构 #

    DFS 依赖于 栈 (Stack) 的思想。当我们用 递归 (Recursion) 来实现 DFS 时，其实就是巧妙地利用了程序的调用栈。

    Go 代码模板 (递归) # 

    // visited 用来记录访问过的节点，防止死循环
var visited = make(map[Node]bool)

func dfs(node Node) {
    // 终止条件 (根据题目定义)
    if node == nil {
        return
    }

    // 标记已访问
    visited[node] = true
    // 处理当前节点逻辑 (例如：打印)
    fmt.Println(node.Value)

    // 递归访问邻居
    for _, neighbor := range node.Neighbors {
        if !visited[neighbor] {
            dfs(neighbor)
        }
    }
}

    什么是广度优先搜索 (BFS)? - “稳扎稳打”的辐射源 #

    BFS 则完全不同，它的策略是 层层推进。想象一下在平静的湖面投下一颗石子，水波纹是如何扩散的？它会先触及最近的一圈，再扩散到稍远的一圈，一圈一圈，非常规律。
